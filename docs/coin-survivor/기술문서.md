# Coin Survivor - Technical Document (Seamless chunk world and natural enemy cluster movements)
## Portfolio Summary
코인세탁소팀에서 코인서바이버 프로젝트에서 담당한 시스템 전체를 하나의 Runtime Flow로 연결한 기술 문서입니다.
- 카메라 이동 중 빈 화면 노출 문제 → Chunk 활성 범위·지연 해제 조정으로 해결
- Chunk 해제 시 드랍 데이터 소실 → DropSaveData 유지, Object만 Pool 반환
- Enemy 생성·등록·반환·처치 보상 흐름 (Factory → RuntimeManager → SpatialGrid → DroppedItemManager)
- Separation 벡터로 Enemy 겹침 완화
- GC Alloc 감소: 컬렉션 재사용, SetTiles 배치 처리, HashSet 중복 검사
* 개발 기록(GitHub Issues): github.com/LoveisChicken/Unity-Bootcamp/issues/64
* [포트폴리오용 팀영상](https://youtu.be/x93I-jS62pk)

## 끊김 없는 Chunk World와 자연스러운 Enemy 군집 이동 구현
> 카메라 이동 중 빈화면이 노출되는 Chunk 로딩 문제와, 대량 Enemy가 한 지점에 겹치는 이동 문제를 해결하기 위해 Chunk Streaming, Obejct Pooling, Separation, SpatialGrid를 연결한 런타임 구조를 설계했습니다.
> **단순 기능 구현이 아니라, Chunk·Enemy·Drop 시스템을 하나의 Runtime Flow로 연결하고 Profiler로 병목을 검증하며 개선한 작업입니다.**

## Proof Section

### Profiler 기반 검증

| 문제      | 개선                                          |결과                                          |
| -------- | ---------------------------------------------- | ---------------------------------------------- |
| Chunk 갱신 중 GC Alloc  | List / HashSet 필드 캐싱 + Clear() 재사용       | 런타임 할당 감소    |
| Tilemap SetTile 반복 호출    | SetTiles 배치 호출    | Tilemap 리빌드 비용 감소   |
| Enemy 중복 등록 검사  | List.Contains → HashSet.Add        |  평균 O(1) 등록 검사  |
| SpatialGrid 주변 탐색 GC   | nearby buffer 재사용     | 탐색 중 할당 감소   |
| Chunk 해제 시 코인 소실 | DropSavaData 유지, Object만 Pool 반환          | 미획득 드랍 복원   |
